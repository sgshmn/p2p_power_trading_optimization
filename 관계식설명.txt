전기 요금
가격을 변할 수 있는 값으로 해서 최적화 문제를 풀자

P = kwh당 가격($/kwh)
U = 전기에너지 사용량(kwh)
O = 전기에너지 생산량(kwh)
C = 비용($)

한명만 생각해 보자
P*U = P*O : 컨슈머가 지불하는 요금이면서 판매자가 얻는 이득
일정 기간(1달? 혹은 1년?) 마다 개인이 판매하는 전기 가격이 바뀐다고 가정하자

k : 한전
p : 프로슈머
c : 컨슈머

Pk = 한전의 전기가격(누진제)
Pp = 개인이 판매하는 전기가격
	한국 : 한달 마다 바뀜
	반데브론 : 일정 kwh까지는 모든 판매자의 금액이 일정(가격 상한제), 1년 단위로 계약(금액이 바뀜)

Uc = 컨슈머가 사용한 전기
Up = 프로슈머가 사용한 전기

Uck = 컨슈머가 한전에게 구매한 전기
Upk = 프로슈머가 한전에게 구매한 전기

Ucp = 컨슈머가 프로슈머에게 구매한 전력량
Upp = 프로슈머가 프로슈머에게 구매한 전기

Ok = 한전이 생산한 총 전력량(무한으로 가정)
Op = 프로슈머가 생산한 총 전력량
	태양광, 풍력, ESS고려해야 함

Cck = 컨슈머가 한전에게 지불한 금액, Uck의 함수다
Ccp = 컨슈머가 프로슈머에게 지불한 금액, 1차 행렬 Ucp의 함수다
Cpk = 프로슈머가 한전에게 지불한 금액, Upk의 함수다
Cpp = 프로슈머가 프로슈머에게 지불한 금액, 1차 행렬 Upp의 함수다


한국

컨슈머가 지불할 금액
Cc = Cck(Uck) + Ccp(Ucp) 여기서 Ccp는 모든 프로슈머에 대해서 고려해야한다(1차 행렬 Ucp -> 스칼라  Ccp 함수)
모든 컨슈머의 소비
모든 c에 대해서 Cc

프로슈머가 지불할 금액
Cp = Cpk(Upk) + Cpp(Upp) - Ccp(Ucp) - Cpp(Upp) 여기서 Upp는 모든 프로슈머에 대해서 고려해야하고, Ccp는 모든 컨슈머에 대해서 고려해야 한다


컨슈머와 프로슈머가 프로슈머에게 지불한 금액 = 프로슈머가 컨슈머와 프로슈머에게 받은 금액 이므로 컨슈머와 프로슈머의 소비를 그냥 합한다면 네 항(Ccp,Cpp,-Ccp,Cpp)은 서로 상쇄된다.

그러면 결과는
Cc + Cp = Cck + Cpk인데 이건 그래프를 그려 볼 것도 없이 한전으로부터 가장 적게 소비하면 최솟값을 갖는다.
즉, 편익 함수를 단순하게 -지불금액으로 하면 안 되고 다른 기준을 갖는 값으로 대체해야한다.
